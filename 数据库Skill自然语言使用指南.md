| 异常检测 | "检测异常" / "有什么异常指标" |

| 风险预测 | "预测一下风险" / "未来会出问题吗" |

> **注意**: 两种报告的区别
> - **巡检报告** (`inspector report`): 健康巡检，关注配置、安全、容量等静态指标，建议定期执行
> - **性能报告** (`diagnose report`): 性能诊断，关注实时性能、慢查询、瓶颈等动态指标，建议按需执行

### 场景3：SQL上线前审核

**你：** 帮我审核一下这条SQL：SELECT * FROM users WHERE email = 'test@test.com'

**我：** 好的，我来全面审核这条SQL。

（自动执行）
- audit sql "SELECT * FROM users WHERE email = 'test@test.com'"
- sql analyze "SELECT * FROM users WHERE email = 'test@test.com'"
- security sql-injection "SELECT * FROM users WHERE email = 'test@test.com'"
- diagnose sql "SELECT * FROM users WHERE email = 'test@test.com'"

**我：** 审核完成！评分75分（C级），发现2个问题：
1. 使用了SELECT *，建议指定具体字段
2. email字段缺少索引，可能导致全表扫描

优化后的SQL：
```sql
SELECT id, name, email FROM users WHERE email = 'test@test.com'
```
建议创建索引：
```sql
CREATE INDEX idx_users_email ON users(email)
```
