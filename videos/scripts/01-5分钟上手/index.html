    视频脚本 01：5 分钟上手 dbskiter¶

    视频信息¶

    • 目标时长：4-5 分钟
    • 目标观众：运维新手、后端开发、无专职 DBA 的中小企业团队
    • 视频风格：终端录屏 + 字幕 + 轻量 BGM（无真人出镜）
    • 录制工具：asciinema / Windows Terminal + OBS / iTerm2 + QuickTime
    • 后期工具：剪映 / CapCut（自动加字幕）

    分镜脚本¶

    片头（0:00-0:15）¶

    画面：黑屏渐出，出现文字 字幕（逐行显示）： 

    dbskiter
数据库运维瑞士军刀
5 分钟上手
    BGM：轻快电子音乐渐入

    第一幕：为什么需要 dbskiter（0:15-0:45）¶

    画面：终端窗口，显示一个"卡顿"的 MySQL 查询（故意放慢动画）

    字幕： 

    业务突然变慢，找不到原因？
没有专职 DBA，数据库问题全靠猜？
每月安全巡检，手动写台账写到崩溃？

    画面切换：显示一行大字 字幕： 

    dbskiter = 诊断 + 监控 + 安全 + 巡检 + 调度 + SQL 审核
一条命令，全自动

    第二幕：安装（0:45-1:15）¶

    画面：干净的终端，黑色背景，命令行输入

    字幕："安装只要一条命令"

    输入： 

    pip install dbskiter

    画面：显示 pip 下载进度条，安装完成提示

    字幕："Python 3.8+ 即可，Windows / Linux / macOS 全支持"

    画面：显示版本号 输入： 

    dbskiter --version

    输出： 

    dbskiter 3.0.22

    字幕："3.0.22 版本，MIT 开源，免费商用"

    第三幕：配置数据库（1:15-2:00）¶

    画面：终端

    字幕："第一次使用？交互式向导帮你配置"

    输入： 

    dbskiter init

    画面：显示交互式提示（逐行高亮）

     ============================================================
   DBSKiter 数据库配置向导 (ENV 格式)
 ============================================================

 请选择数据库类型：

   1. MYSQL        (默认端口: 3306)
   2. POSTGRESQL   (默认端口: 5432)
   3. ORACLE       (默认端口: 1521)
   4. MSSQL        (默认端口: 1433)
   5. CLICKHOUSE   (默认端口: 8123)
   6. SQLITE       (默认端口: 0)

 请输入序号 (1-6): 1

 第一个数据库将作为默认配置（无前缀）
  数据库别名（直接回车使用 'default'）：jump

 --- MYSQL 连接配置 ---

   主机地址 [数据库主机地址]: 127.0.0.1
   端口 [3306]: 3306
   用户名 [root]: root
   密码:
   数据库名 [test]: jump

 是否继续添加更多数据库? [y/N]: N

 配置已保存到: /tmp/.env

    字幕："不用手写配置文件，问答式向导，2 分钟搞定"

    画面：显示配置文件内容（快速闪过，不细看）

    字幕："支持多数据库配置，生产、测试、开发环境分开管理"

    第四幕：第一条命令：健康检查（2:00-2:45）¶

    画面：终端

    字幕："先看看数据库健康状态"

    输入： 

    dbskiter --database=jump monitor health

    画面：输出结果，用颜色高亮（终端自动着色）

    健康评分: 66.4/100

[绿色] 连接数使用率: 2.6%   正常
[绿色] 缓冲池命中率: 99.90% 优秀
[黄色] 临时表磁盘使用率: 61.96% 警告
[黄色] 行锁等待次数: 147 警告
[红色] 字符集: 10 张表使用旧版 UTF8 严重

    字幕："绿色 = 正常，黄色 = 警告，红色 = 严重 一目了然，不用看 raw SQL"

    画面：用箭头图标指向黄色和红色项

    第五幕：一键巡检生成报告（2:45-3:30）¶

    画面：终端

    字幕："月度巡检？一条命令出报告"

    输入： 

    dbskiter --database=jump inspector report --output report.html

    画面：显示命令执行过程，文件生成提示

    正在巡检... [====================] 100%
报告已生成: report.html
包含: 45 项检查、3 个警告、1 个严重

    画面：切换到浏览器，打开 report.html（录屏软件切窗口）

    画面展示： - 顶部：健康评分大数字（66.4） - 雷达图：配置、性能、安全、存储、容量 5 维度 - 表格：检查项列表，带颜色标记 - 底部：优化建议清单

    字幕："HTML 报告直接发给领导，不用手动写台账"

    第六幕：安全审计（3:30-4:00）¶

    画面：终端

    字幕："安全合规？一键扫描"

    输入： 

    dbskiter --database=jump security audit

    画面：输出结果，高亮 WARNING

    [通过] 无空密码用户
[通过] 无匿名用户
[通过] SSL 已启用
[警告] 4 个 SUPER 权限用户
[警告] 1 个远程 root 访问

    字幕："SUPER 权限、远程 root、弱密码、SQL 注入风险 全自动扫描，不用手动跑 SQL"

    画面：切换到 Trae IDE（录屏切窗口）

    帮我检查 jump 数据库的健康状态

    "jump 数据库有没有死锁？"

    录制技术说明¶

    工具选择¶

    平台 推荐工具 理由
    Windows Windows Terminal + OBS Studio 终端美观，OBS 免费专业
    macOS iTerm2 + QuickTime 终端支持配色，QuickTime 自带录屏
    Linux asciinema 终端原生录制，文件极小，可嵌入网页
    跨平台 asciinema 输出 .cast 文件，支持网页播放器嵌入

    终端配色建议¶

    // Windows Terminal 配色（推荐）
{
  "background": "#0C0C0C",
  "foreground": "#CCCCCC",
  "cursorColor": "#FFFFFF",
  "green": "#13A10E",
  "yellow": "#C19C00",
  "red": "#C50F1F",
  "blue": "#0037DA"
}

    字体建议¶

    • 推荐：Cascadia Code、JetBrains Mono、Fira Code
    • 字号：18-20px（录屏时保证可读性）

    后期处理¶

    1. 剪映 / CapCut
    2. 导入录屏文件
    3. 使用"自动识别字幕"功能
    4. 调整字幕位置：底部居中，不遮挡命令行

    5. 导出 1080p 60fps

    6. GIF 版本（用于 README）

    7. 使用 ffmpeg 提取关键片段（安装、第一条命令、巡检报告）
    8. 转换为 GIF（限制 5-10 秒，文件 < 5MB）
    # ffmpeg 提取片段并转 GIF
ffmpeg -i video.mp4 -ss 00:00:45 -t 5 -vf "fps=10,scale=1280:-1:flags=lanczos" output.gif
